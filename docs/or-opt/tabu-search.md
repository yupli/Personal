# 禁忌搜索（Tabu Search, TS）

禁忌搜索在**邻域搜索**框架上引入**短期记忆（禁忌表）**：将近期执行过的「移动」或其**属性**（如被交换的边、被翻转的变量下标等）列为**禁忌**，在一定步数内禁止立即回退，以避免在局部最优附近来回震荡。可配合**渴望准则**（aspiration）在禁忌移动仍带来历史最优时破禁。属于单点、带记忆的元启发式，见 [元启发式总览](metaheuristics.md)。

**适用**：组合优化（TSP、调度、图着色等）的近似求解；邻域与**移动属性**的定义对效果影响大。

**核心**：每步在（经渴望过滤后的）**可接受邻域**中选使目标改进或「最不坏」的移动；禁忌表按** tenure**（禁忌步长）滑动更新，常取近优解、多邻域与重启。

---

## 算法（最小化示意）

以目标函数 f(s) 最小化叙述；禁忌对象记为与**移动**关联的**属性** a（实现上常用属性集合 A，对选中的移动更新 A）。

::: algorithm 禁忌搜索主循环（单解、邻域 + 禁忌表）

### 1. 初始化
选定初始解 s（随机或启发式），记历史最优 s⋆ ← s。初始化**空**禁忌结构（如记录最近禁止的属性及剩余禁忌步数），设定参数：邻域算子、禁忌步长（tenure）、最大迭代、无改进步数等。

### 2. 邻域与筛选
当前解为 s。由邻域算子生成候选集 N(s)（或采样子集）。对每个候选移动 m：得到 s′ = m(s)，计算 f(s′)。若 m 对应属性在禁忌期内，则 m 一般**不可选**，除非满足**渴望准则**（例如 f(s′) < f(s⋆)，即破禁能带来新的历史最优）。

### 3. 选择与移动
在**可接受**的候选中，按预定规则选取一步（常见：若存在改进移动则选最优改进；否则选**最小增**的恶化移动以离开局部）。执行该移动，更新 s。

### 4. 更新禁忌与全局最优
将该步所涉**属性**记入禁忌表，并赋予剩余禁忌步数为 tenure（常可随迭代或随机微扰）。若 f(s) < f(s⋆)，令 s⋆ ← s。

### 5. 终止
达到迭代/时间上限、或长期无改进（带可能的重启策略）等，则停；输出 s⋆。

:::

---

## 关键要素

- **移动与属性**：禁忌可针对「边」、「位置对」、「变量_flip」等；需与邻域定义一致，否则表意不清。  
- **Tenure**：过短则仍易循环；过长则过度限制探索。常与问题规模、邻域大小联调。  
- **渴望准则**：典型为「优于当前历史最优则破禁」，避免错过好的解。  
- **长期策略**（可选）：频率惩罚、强化/分散、多邻域切换等，在此不展开。
