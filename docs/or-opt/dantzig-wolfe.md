# Dantzig-Wolfe 分解算法流程

## 1. 准备阶段：模型重构

首先，需要将**原问题（Original Problem）**改造成**主问题（Master Problem）**。

**输入**：
- 原问题的约束拆分为"公共约束 $Ax \leq b$"和"局部约束 $Bx \leq d$"

**变换**：
- 将变量 $x$ 用局部约束集合的极点 $x^j$ 的凸组合表示：
  $$x = \sum_j \lambda_j x^j, \quad \text{其中 } \sum_j \lambda_j = 1, \; \lambda_j \geq 0$$

**目标**：
- 把这个表示式代入原目标函数和公共约束，得到只含有权重 $\lambda_j$ 的模型（即主问题）

---

## 2. 第一阶段：寻找初始可行基（两阶段法）

算法不能凭空开始，需要至少一个极点来启动。

**引入人工变量**：
- 如之前看到的，在 RMP 的公共约束中加入人工变量 $x_a$
- 目标函数设为 $\min \sum x_a$

**迭代循环**：
- 按照列生成的逻辑不断加入新列

**结束标志**：
- 如果人工变量的最小值达到了 0，说明找到了一个（或一组）能够满足所有约束的初始极点
- 删掉人工变量，正式进入第二阶段

---

## 3. 第二阶段：核心迭代循环（列生成）

这是 DW 算法最精彩的部分，它是一个在**受限主问题（RMP）**和**子问题（Pricing Subproblem）**之间迭代的过程：

### 3.1 第一步：求解当前 RMP

解当前的 RMP 模型，获得两个关键的对偶值（Dual Values）：

- **$\pi$**：对应公共约束（Linking Constraints）的对偶价格
- **$\sigma$**：对应凸组合约束（$\sum \lambda = 1$）的对偶值

### 3.2 第二步：求解子问题（Pricing）

利用拿到的 $\pi$，构造子问题的目标函数：

$$\min \bar{c} = (c - \pi A) x - \sigma$$

在局部约束 $Bx \leq d$ 内求解这个子问题。这个过程是在寻找：**有没有哪一个极点 $x^{new}$，能够让主问题的目标函数进一步下降？**

### 3.3 第三步：判断与入基

**如果子问题的最优值 $\bar{c} < 0$**（以最小化为例）：
- 说明找到了一个"好方案"！
- 将这个极点 $x^{new}$ 转化成 RMP 的一列新数据（即一个新的变量 $\lambda_{new}$）
- 返回第一步继续迭代

**如果子问题的最优值 $\bar{c} \geq 0$**：
- 说明在当前的资源价格 $\pi$ 下，已经没有任何新方案能让成本更低了
- 算法收敛，达到全局最优

---

## 4. 结果还原：回到物理变量

当拿到了最优的 $\lambda_j^*$ 后，最后一步是把这些权重重新加权到对应的极点上，还原回原问题的变量：

$$x^* = \sum_j \lambda_j^* x^j$$

这就是最初想要的 $x_1, x_2, \ldots$ 的具体数值。
