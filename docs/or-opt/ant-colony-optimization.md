# 蚁群优化（Ant Colony Optimization, ACO）

蚁群优化以「信息素 + 概率构造」为骨架：在图中（常为有向图、TSP 完全图等）令蚂蚁按与信息素浓度及启发式（如边权倒数）相关的概率逐步构造可行解，再用所得解质量反馈更新信息素；挥发项避免无界累积。与遗传算法等群体方法相比，更强调在图结构上构造路径与在边上的协作学习。见 [元启发式总览](metaheuristics.md)。

**适用**：可表述为在图上逐步决策的组合问题（TSP、路由、排程、子集等），且能定义合理的启发式可见度 η 与放缩参数 α、β；对大规模问题常控制蚂蚁数量与每步候选规模以控计算量。

**核心**：多轮「若干蚂蚁各自构造解 → 评价 → 按规则对路径上或边上的信息素作增量与乘性挥发」。解越好，对所用弧（或决策）的信息素增加越多，后续蚂蚁更倾向重走，形成正反馈；挥发则防止早期路径垄断全体。

---

## 算法（思想性示意、以弧上信息素为例）

用边 e 上信息素 τ_e≥0 与可见度 η_e（如 TSP 中 η_ij = 1/d_ij，d 为距离）。下框为单轮迭代的抽象流程；各具体 ACO（AS、MMAS、ACS 等）在蚂蚁条数 m、信息素上界/下界、局部/全局更新顺序上不同，在实现中定稿即可。

::: algorithm 蚁群优化 一轮（多蚂蚁构造 + 信息素更新）

### 1. 初始化
若首轮，按问题设定 τ 初值（常数或小随机）；η 由实例预计算。准备 m 只蚂蚁，自源点或 TSP 中规定起点出发（多起点策略依实现定稿）。

### 2. 蚂蚁构造解
对每只蚂蚁在约束下按随机成比例规则逐步延长路径：在下一步可选弧集合中，以正比于 τ_e^α · η_e^β 的概率选取下一弧，直至得到完整可行解 s；评价 f(s)。对每只蚂蚁的个体极值、群体极值 s⋆ 按实现更新。

### 3. 信息素挥发
对全体相关弧 e，τ_e ← (1 − ρ) τ_e，标量 ρ∈(0,1) 为挥发率；若某变体对 τ 有上/下界，在挥发后截断到允许区间（如 MMAS）。

### 4. 信息素增量
按本轮蚂蚁构造的路径或仅取最优解对应路径，对路经弧 e 上 τ_e 加 Δτ_e。Δ 常与 1/f(s) 或单调变换有关；ACS 等可能另含局部、全局双层更新，此处不展开为单一公式。

### 5. 终止
若已满足外轮总迭代、时间、无改进等则停，输出 s⋆；否则回到步骤 2 进入下一轮（蚂蚁重新构造时可共用更新后的 τ）。

:::

## 关键要素

**α 与 β**：α 大则更信信息素（强化历史路径），β 大则更信启发式（贪心可见度强）；与 ρ、m 共同决定探索—利用折中。

**初值与边界**：过低的 τ(0) 让早期过随机；MMAS 等对 τ 设 min/max 以缓解早收敛和停滞。参数需可复现地写在实验设置里。

**与问题建图**：决策若是「下一点」而非边，可改为结点对上的信息素或等价权重；多商品、时间窗等需在构造步的候选集中编码约束，否则不可行解爆炸。

**代表变体**：蚁群系统（AS）、带精英策略、MMAS、ACS 等在挥发与放量的分工上不同；选哪一种与实现细节取决于规模与可维护性，本页不比较优劣数值。
