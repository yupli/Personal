# 变邻域搜索（Variable Neighborhood Search, VNS）

变邻域搜索由 Mladenović 与 Hansen 等系统化提出：在单点搜索框架下，预备一组由弱到强（或由近到远）的邻域结构，在局部最优处通过「换邻域」或「抖动 + 再局部搜索」跳出当前吸引域。与固定单邻域的下降法相比，显式利用多邻域的互补性，属单解、基于邻域的元启发式，见 [元启发式总览](metaheuristics.md)。

**适用**：组合优化（路径、调度、划分等）与部分连续问题在能定义多个邻域算子且每步评价 f 可承受时；邻域设计对效果起决定作用。

**核心**：当前解在某一邻域内做局部搜索直至局部最优后，若不能改进，则按规则切换到更大扰动范围的邻域（抖动）或换一套邻域结构再搜；成功改进则往往回到最小邻域继续细化。常与变邻域下降（VND）等子过程组合。

---

## 记号与邻域结构

设目标 f(s) 最小化，s 为可行解。给定有限个邻域映射 N_k，k∈{1,…,K_max}，通常按「步幅」或「破坏程度」单调：N_1 最局部，N_Kmax 扰动最大。对点 s，N_k(s) 表示经第 k 类算子一步可达的邻域点集（实现上常从 N_k(s) 中采样或枚举子集）。

---

## 典型子过程：变邻域下降（VND）

VND 假定已有多邻域 N_1,…,N_K，无随机抖动，只做确定性局部最优传递。

::: algorithm 变邻域下降 VND

### 1. 初始化
选定可行解 s，设 k ← 1。

### 2. 在当前邻域内找改进
在 N_k(s) 中寻找使 f 下降的点 s′（依「首次改进」「最速下降」等规则，直到 N_k 内对 s 为局部最优）。

### 3. 邻域切换
若找到改进 s′，令 s ← s′，k ← 1（回到最细邻域继续）；若 N_k 内无改进，令 k ← k+1。

### 4. 终止
若 k>K（所有 N_k 对当前 s 均局部最优），停，输出 s；否则回到步骤 2。

:::

VND 本身仍可能停在多邻域意义下的强局部优；基本 VNS 在此外层增加随机抖动以跳离。

## 基本 VNS 主循环（最小化示意）

在 VND 或「单邻域下降」内层之外，用随机抖动从当前邻域的吸引域中抽出候选，再局部搜索，形成探索—利用交替。

::: algorithm 基本 VNS（单解、多邻域 + 抖动）

### 1. 初始化
生成初始解 s，记历史最优 s⋆ ← s。设邻域族 N_1,…,N_K 与最大抖动强度 K_max。可选：内层用 VND 或固定顺序单邻域下降。

### 2. 外层邻域下标
设 k ← 1，表示当前使用的抖动/搜索邻域级别。

### 3. 抖动（Shaking）
在 N_k(s) 中随机取一点 s′（或沿某随机步数在 N_k 中游走），使 s 离开当前局优吸引域的邻域子集；s′ 不必优于 s。

### 4. 局部搜索
以 s′ 为起点，在全部或部分邻域上执行 VND/下降，得局部最优点 s″。

### 5. 接受与邻域级更新
若 f(s″)<f(s)，令 s ← s″，k ← 1，若 f(s″)<f(s⋆) 则 s⋆ ← s″；否则 k ← k+1。若 k>K_max，可 k ← 1 并配合全局迭代上限或 s ← s″ 的变体，依实现定稿。

### 6. 终止
达时间、总评价次数、连续无改进步数等，输出 s⋆。

:::

---

## 关键要素

**邻域族顺序**：k 小对应小扰动、利于微调；k 大利于跳出，但过强抖动易损可行或需修复算子。实践上常问题特化设计多种 2-opt、insert、swapping、大规模破坏—修复等，并在 N_k 间保持可实现的评价代价。

**抖动与内层下降分工**：抖动负责随机性，内层负责把抖动结果拉到新的局优。重复「抖—搜」比单邻域上长链随机游走更易控。

**与禁忌搜索、模拟退火**：VNS 用「多邻域结构 + 显式改变搜索邻域/抖动范围」管理探索，不必依赖禁忌表或温度；也可与 TS/SA 混合为同一内层或外层。

**扩展**：如简化 VNS（reduced VNS）限制抖动与搜索子集、并行 VNS、及在全局框架中与精确子程序、大规模邻域搜索衔接，均属实现与文献变体，本页不逐条列参。
