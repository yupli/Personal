# 遗传算法（Genetic Algorithm, GA）

遗传算法在**有限规模种群**上模拟「选择—交叉—变异」的循环：适者更易留下后代，**交叉**组合父代片段，**变异**引入小扰动。适合**有良好编码**与**可快速评价适应度**的问题，是群体类元启发式的代表。见 [元启发式总览](metaheuristics.md)。（更广义的**进化计算**还包含进化策略、遗传规划等，本页以经典 GA 为主干。）

**适用**：组合或参数优化在能定义**可行编码**、**适应度/目标 f** 且邻域**不宜**用单点穷举时；规模很大时常与**局部搜索、修复**混合。

**核心**：在一代代**繁殖—替换**中，以选择压推动目标改进、以交叉+变异在搜索空间**扩散与补多样性**；常保留**历史最优**（精英或外部存档）以免好解在随机操作中丢失。

---

## 算法（最小化示意）

以最小化 f(s) 为例；s 在实现中为**个体编码**（二进制串、排列、实数向量等），适应度可**直接取** f(s) 或取单调变换（如 1 / (1+f)），**评价规则**在全程保持一致。

::: algorithm 遗传算法主循环（种群 + 选择 / 交叉 / 变异）

### 1. 初始化
在可行域上随机或启发式生成初始种群 P（个体数为 N），逐个评价 f。记当前种群里最优为 s⋆；设定：选择方式、交叉与变异算子、交叉率/变异率、代际替换策略、最大代数或评价次数、可选精英保留比例等。

### 2. 一代迭代（重复直至终止条件）
- **亲本选择**：由当前 P 中按选择规则选出一批用于繁殖的**父本**（可重复选同一好个体，依锦标赛、轮盘、排序等算子）。  
- **重组（交叉）**：对父本两两或按算子**配对**，以**交叉率** p_c 在编码上执行交叉，得**子代**中间集。无交叉时子代可视为父本复制。  
- **变异**：对子代个体以**变异率** p_m 执行变异（如比特翻、对换、高斯微扰等），**保持或修复**可行性。  
- **评价与替换**：计算子代 f；按「完全替换、稳态（只换一部分）+ 精英」等策略**组成**下一代 P；若出现更优个体则更新 s⋆。

### 3. 终止
达到预设代数/时间/总适应度评价次数、或 s⋆ 长期不改进等，则停；输出 s⋆。

:::

## 关键要素

- **编码与可行性**：TSP 常用**排列**；背包用**0–1 串**；实优化用**实向量**。非法个体需**修复、惩罚**或**专用交叉变异**保证可解。  
- **选择压**：过强易早熟、种群趋同；过弱则收敛慢。常配合**精英**保留当前最优。  
- **交叉 / 变异**：与编码强相关；**PMX、OX** 等用于排列，**单点/多点交叉** 用于二进串，**算术交叉、SBX** 等用于实数。  
- **参数**：N、p_c、p_m 及终止条件多靠**调参**与多次运行。  

## 与同类方法的差异（提要）

- 相对**单点**方法（[模拟退火](simulated-annealing.md)、[禁忌搜索](tabu-search.md)）：**并行**维护多解、用**种内重组**在解空间**跳跃**，不必每步都定义邻域，但**编码设计**是成败关键。  
- 常与**局部搜索**（Memetic）、**多目标**（NSGA-II 等）结合，本页不展开。
