GitHub Pages

你看到的是「仓库说明」，不是 VitePress 站点

若 Settings → Pages 仍从 main 分支的根目录发布，GitHub 会用 Jekyll 把根目录的 README.md 当成主页，所以会像「只有文字、没有导航和首页布局」——这与 笑笑的站 （从 gh-pages 发布构建产物）不一样。

请把发布源改成构建后的静态分支：

  1. 打开仓库 Actions，确认 Deploy VitePress 已成功（会更新 gh-pages）。
  2. Settings → Pages → Build and deployment：Source 选 Deploy from a branch，分支选 gh-pages，目录选 / (root)，保存。
  3. 等待一两分钟后访问 https://yupli.github.io/Personal/，应出现带顶栏与首页的文档站。

另请确认 Settings → Actions → General → Workflow permissions 为 Read and write，否则无法推送 gh-pages。
